# Contributing

Contributions to improve this documentation are welcome.

## Quick Edits

1. Click the pencil icon on any page in GitHub
2. Make your changes
3. Submit a pull request

## Larger Changes

1. Fork the repository
2. Clone and set up locally (see [Testing Locally](docs/developers/testing-locally.md))
3. Create a branch: `git checkout -b my-changes`
4. Make changes and test with `mkdocs serve`
5. Commit and push
6. Open a pull request

## Guidelines

- Keep language clear and accessible
- Use tables for structured data
- Place images in `docs/assets/`
- Update `mkdocs.yml` navigation when adding pages

## Questions

Open an issue on GitHub.
